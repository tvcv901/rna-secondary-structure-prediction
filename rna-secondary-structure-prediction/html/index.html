RNA Secondary Sequence Prediction

Problem Statement

We are given a RNA sequence which is a string consisting of the characters in the set of nitrogenous bases \(\{A, C, G, U\}\). The task is to predict the secondary RNA structure for the given RNA sequence which also satifies the following constraints:

  1. \(A\) can form a pair only with \(U\), and \(C\) can form a pair only with \(G\)
  2. There should be no knots in the secondary structure (two base pairs cannot overlap or cross each other)
  3. There should be no kinks in the secondary structure (two bases cannot form a pair if they are close to each other. In this problem, the bases cannot form a pair if there are less than 5 between them)
  4. Each base can form a pair at most once (the same base cannot be a part of two different pairs)

Algorithm

The optimal solution to the above problem uses a dynamic programming approach. Let \(dp(i, j)\) be the maximum number of base pairs that can be formed without violating the above constraints in the range \([i, j]\), where \(1 \le i, j \le len(sequence)\).

The solution for \(dp(i, j)\) is given as:

\[dp(i, j) = \begin{cases} 0 & i \ge j - 4 \\ \max \begin{cases} dp(i, j - 1), \\ \underset{\forall t \in [i, j - 1]}\max \{dp(i, t - 1) + dp(t + 1, j - 1) + 1\} \end{cases} & otherwise \end{cases}\]

The solution to the overall problem is finding the maximum number of base pairs for the given RNA sequence, which is \(dp(1, len(sequence))\).

Understanding the Equation

Suppose we are trying to find the maximum number of base pairs that can be formed in the interval \([i, j]\).

From the figure, suppose the bases at positions \(t (t \in [i, j - 1])\) and \(j\) can be paired. The problem \([i, j]\) gets split into two subproblems because of the pairing.
The first subproblem (highlighted in orange) is for the interval \([i, t - 1]\), whose solution is \(dp(i, t - 1)\), and
The second subproblem (highlighted in yellow) is for the interval \([t + 1, j - 1]\), whose solution is \(dp(t + 1, j - 1)\).

Hence, \(dp(i, j)\) will be the sum of (assuming that pairing \(t\) and \(j\) gives the maximum):

  • \(dp(i, t - 1)\) (the first subproblem)
  • \(dp(t + 1, j - 1)\) (the second subproblem)
  • \(1\) (for the pair \((t, j)\))

Implementation

Each RNA sequence has a matrix and an array of base pairs associated with it. We use a matrix \(maxMatchings\) of size \(n\) x \(n\), where \(n\) is the length of the RNA sequence. \(maxMatchings[i][j]\) is the maximum number of base pairs that can be obtained in the interval \([i, j]\) without violating the constraints. This matrix is used to get the array of base pairs of the secondary structure of RNA.

Complexity

Let \(T(n)\) be the running time to solve the problem whose RNA sequence is of length \(n\). We have the recurrence relation:
\(T(n > 5) = \sum_{t = 0}^{n - 2}T(n - 2 - t) * T(t)\),
\(T(n \le 4) = c\)
On solving the recurrence relation, we get that \(T(n)\) is exponential with \(n\).

Notice that we solve the same subproblem multiple times if the recursive approach is used. To avoid this, we can store the answer to the subproblem once it has been solved. If we come across the same subproblem again, we can obtain it directly from the stored values, instead of solving the subproblem again. This step is known as memoization.

Order of Subproblems

Suppose we have the problem in the interval \([i, j]\). The solution for this problem depends on subproblem(s) of size(s) smaller than the interval \([i, j]\).
Hence, in order to solve a bigger problem, we need to solve the problems of smaller sizes first. Therefore, we need to solve the subproblems of smallest size first (starting with size \(6\) for this problem). Following the above ordering to solve the subproblems and using memoization, the time complexity of the solution becomes \(O(n^3)\).

For memoization, we are storing the answer for each subproblem \([i, j]\) \(1 \le i \le j \le n\). The total number of subproblems can be calculated as \(\sum_{i = 1}^{n} \sum_{j = i}^{n} 1\), which is \(O(n^2)\). Hence, the space complexity for the solution is \(O(n^2)\).
